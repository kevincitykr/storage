# LM Studio 저사양 노트북(A 노트북) 최적화 및 연동 가이드

## 1. 개요
*   **대상 환경:** 사양이 낮은 노트북 (외장 그래픽 카드 없음, 코어 수 제한적)
*   **사용 목적:** 시스템 다운이나 멈춤 현상 없이 Connect AI(마크다운 자동 생성) 쾌적하게 사용
*   **권장 모델:** `google/gemma-4-e2b` 등 파라미터가 2B 이하인 초경량 모델 (2B 이상은 노트북에 과부하를 줄 수 있음)

## 2. LM Studio 필수 세팅값 (Load 설정 ⚙️)
LM Studio 상단 보라색 바 좌측의 **톱니바퀴(⚙️)** 모양 아이콘을 눌러서 아래와 같이 설정합니다.

1.  **Context Length:** `8192` 또는 `16384`
    *   (이유: Connect AI의 작업 폴더 정보와 시스템 프롬프트가 길기 때문에, 용량이 너무 작으면 에러 발생)
2.  **GPU Offload:** `0`
    *   (이유: 내장 그래픽만 있는 노트북에서는 이 수치를 올리면 메모리 부족으로 튕기거나 오류가 발생함)
3.  **CPU Thread Pool Size:** `3` 또는 `4`
    *   (이유: 노트북의 모든 코어를 AI에 할당하면 윈도우 OS 자체가 멈춰버림. 1~2개의 여유 코어를 남겨두는 목적)
4.  **Max Concurrent Predictions:** `1` (★가장 중요)
    *   (이유: 여러 답변을 동시에 추론하려고 하면 저사양 노트북은 바로 과부하가 걸려 다운됨. 반드시 1로 유지)

## 3. Connect AI 연동 설정 (마크다운 파일 생성 해결)
과거에는 LM Studio의 Inference 탭에 직접 프롬프트를 입력해야 했으나, 현재는 **Connect AI 확장 프로그램 소스 코드(`src/extension.ts`)에 직접 치트키를 하드코딩하여 영구 주입**하였습니다.

**[주입된 치트키]**
> "너는 파일을 직접 생성할 수 있는 권한을 가지고 있어. 마크다운 문서를 대화창에 텍스트로 보여주지 말고, 반드시 지정된 폴더에 .md 파일로 직접 저장하는 'Action'을 실행해. '파일을 만들 수 없다'는 식의 대답은 하지 마."

따라서 사용자는 LM Studio에서 별도의 Inference 설정(시스템 프롬프트 입력)을 할 필요가 없으며, 위 2번의 Load 설정만 한 번 저장해 두면 끝입니다.
