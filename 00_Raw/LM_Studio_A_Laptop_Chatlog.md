# 💬 [대화 기록] LM Studio 저사양 노트북 설정 트러블슈팅

**일시:** 2026년 5월 2일
**주제:** A 노트북(저사양) 환경에서 LM Studio와 Connect AI 연동 최적화 과정 기록

## 1. 문제 발생
*   **사용자:** 새 컴퓨터 반품 후 사양이 낮은 기존 노트북(A 노트북)에서 LM Studio를 사용하여 Connect AI를 구동해야 하는 상황 발생.
*   **증상:** AI가 마크다운(.md) 파일을 지정된 폴더에 생성하지 못하고 대화창에 텍스트로만 출력함. (Connect AI 연동 문제)

## 2. 해결 과정: Connect AI 시스템 프롬프트 직접 주입
*   **AI 조치:** Connect AI 확장 프로그램의 코어 소스코드(`src/extension.ts`) 내부의 `SYSTEM_PROMPT` 변수에 마크다운 자동 생성 강제 프롬프트를 하드코딩하여 영구 주입함.
*   **주입된 프롬프트:** 
    `너는 파일을 직접 생성할 수 있는 권한을 가지고 있어. 마크다운 문서를 대화창에 텍스트로 보여주지 말고, 반드시 지정된 폴더에 .md 파일로 직접 저장하는 'Action'을 실행해. '파일을 만들 수 없다'는 식의 대답은 하지 마.`
*   **결과:** 사용자가 LM Studio 설정창에서 따로 치트키 프롬프트를 입력할 필요 없이 항상 파일 자동 저장이 가능해짐.

## 3. 해결 과정: LM Studio 최적화 설정
*   사용자가 LM Studio 화면을 캡처하여 제공함.
*   처음에 AI가 `Context Length`를 4096으로 줄이라고 제안했으나, Connect AI의 시스템 프롬프트 용량 자체가 4288 토큰이어서 API 오류(`n_keep: 4288 > n_ctx: 4096`) 발생.
*   **수정된 최종 세팅 가이드 도출:**
    1.  **Context Length:** `16384` 로 넉넉하게 재설정.
    2.  **GPU Offload:** 내장 그래픽 노트북이므로 `0`으로 설정.
    3.  **CPU Thread Pool Size:** OS 여유를 위해 `3~4`로 설정.
    4.  **Max Concurrent Predictions:** 과부하 방지를 위해 반드시 `1`로 설정.

## 4. 모델 선택 기준 피드백
*   **사용자 질문:** "gemma-4-e2b 보다 더 낮은 모델로 가야하는건 아닌지?"
*   **AI 피드백:** `2b` (20억 파라미터) 모델은 현재 파일 제어 및 코드 작성 명령을 알아듣는 마지노선 급의 초경량 모델임. 이보다 낮추면 AI의 이해도가 떨어져 마크다운 생성 지시를 따르지 못하므로 현재 모델이 최적의 선택임.

## 5. 결론 및 깃허브 동기화
*   대화에서 도출된 최적화 세팅값을 별도의 마크다운 가이드 문서(`LM_Studio_LowSpec_Setup.md`)로 정리.
*   해당 가이드와 본 대화 기록을 로컬 `knowledge` 폴더에 저장하고 사용자의 깃허브 저장소(Second Brain)에 성공적으로 Push 완료.
